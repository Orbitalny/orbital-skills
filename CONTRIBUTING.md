# Contributing to orbital-skills

Thanks for contributing. This project values clear thinking, reusable patterns, and practical outcomes.

## What to contribute

- New skill files (`*.md`) that are reusable and high signal
- Improvements to existing skills (clarity, precision, edge cases)
- Better docs, examples, and onboarding

## Skill format

Every skill should include frontmatter and these sections:

- `Core Standard`
- `How to Think`
- `How to Execute`
- `How to Finish`

Start from [`SKILL_TEMPLATE.md`](./SKILL_TEMPLATE.md).

## Quality bar

Before opening a PR, verify the skill:

- is specific about when to trigger
- avoids generic motivational language
- produces concrete, actionable output
- has no contradictory instructions
- is useful beyond a single niche prompt

## Pull request process

1. Fork the repo and create a feature branch.
2. Add or edit skill files.
3. Update `README.md` skill table if you add a new skill.
4. Open a PR using the template.
5. Respond to review feedback quickly and constructively.

## Commit style

Use clear commit messages, e.g.:

- `feat(skill): add research-sprint skill`
- `docs: clarify quick start`
- `fix(skill): tighten trigger conditions`

## Community expectations

By participating, you agree to the [`CODE_OF_CONDUCT.md`](./CODE_OF_CONDUCT.md).

Thanks for building this with us.
