# Security Policy

## Reporting a vulnerability

If you find a security issue related to repository automation, workflows, or published assets:

1. Do **not** open a public issue.
2. Create a private security advisory in GitHub, or contact the maintainer directly.
3. Include reproduction steps, impact, and suggested mitigation if available.

We aim to acknowledge reports within 72 hours and provide remediation guidance quickly.

## Supported versions

This repository is documentation-first. Security-sensitive changes apply to the latest default branch.
